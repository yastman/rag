# 📜 Описание скриптов проекта

Этот документ содержит описание всех утилитарных скриптов в корне проекта.

---

## 🔌 Скрипты тестирования подключения к Qdrant

### `test_basic_connection.py`

**Назначение**: Базовый тест подключения к Qdrant без использования сторонних зависимостей (кроме стандартной библиотеки Python).

**Что делает**:
- ✅ Проверяет доступность Qdrant через HTTP API
- ✅ Получает версию и commit Qdrant
- ✅ Выводит список всех коллекций
- ✅ Показывает детальную информацию о первой коллекции (количество точек, векторов, конфигурацию)

**Когда использовать**:
- Первый запуск для проверки, что Qdrant доступен
- Отладка сетевых проблем
- Не требует установленных зависимостей проекта

**Запуск**:
```bash
python test_basic_connection.py
```

**Зависимости**: Только стандартная библиотека Python (urllib, json)

---

### `test_qdrant_connection.py`

**Назначение**: Тест подключения к Qdrant с использованием официального клиента `qdrant-client` и конфигурации проекта.

**Что делает**:
- ✅ Загружает настройки из `src.config.settings`
- ✅ Подключается к Qdrant через `QdrantClient`
- ✅ Выводит список коллекций
- ✅ Показывает детали каждой коллекции (количество точек, размерность векторов, метрику)

**Когда использовать**:
- Проверка корректности настроек проекта (`.env`)
- Тестирование после изменения конфигурации
- Проверка, что `qdrant-client` установлен и работает

**Запуск**:
```bash
python test_qdrant_connection.py
```

**Зависимости**:
- `qdrant-client`
- `src.config.settings`

---

### `test_qdrant_read.py`

**Назначение**: Расширенный тест чтения данных из Qdrant. Проверяет возможность получения и фильтрации данных.

**Что делает**:
- ✅ Подключается к Qdrant
- ✅ Получает список коллекций
- ✅ Читает первые 3 точки из коллекции (scroll)
- ✅ Выводит payload каждой точки (document_name, article_number, chapter, text)
- ✅ Тестирует фильтрацию по полю `article_number` (ищет статьи с номером "1")

**Когда использовать**:
- Проверка, что данные в коллекции корректны
- Тестирование фильтрации и поиска
- Отладка payload структуры
- Проверка, что индексация прошла успешно

**Запуск**:
```bash
python test_qdrant_read.py
```

**Зависимости**:
- `qdrant-client`
- `python-dotenv`

**Пример вывода**:
```
📌 Точка 1 (ID: 00250285-4464-1022-9f2a-c6ffd344d3f3):
  document_name: Кримінальний кодекс України
  article_number: 25
  text: караються обмеженням волі на строк від трьох до п'яти років...
```

## 🔄 Порядок использования скриптов

Рекомендуемая последовательность для проверки системы:

1. **`test_basic_connection.py`** - Базовая проверка доступности Qdrant
2. **`test_qdrant_connection.py`** - Проверка конфигурации проекта
3. **`test_qdrant_read.py`** - Проверка данных в коллекциях

## ⚠️ Устранение неполадок

### Ошибка "Connection refused"
```bash
# Проверьте, запущен ли Qdrant
docker ps | grep qdrant

# Запустите Qdrant
docker compose up -d qdrant
```

### Ошибка "API key invalid"
```bash
# Проверьте .env файл
cat .env | grep QDRANT_API_KEY

# Убедитесь, что API key корректен
```

### Ошибка "Collection not found"
```bash
# Проверьте список коллекций
python test_qdrant_connection.py

# Возможно, нужно проиндексировать документы
# (см. основной README.md)
```

## 📝 Примечания

- Все скрипты возвращают код выхода `0` при успехе и `1` при ошибке
- Скрипты можно использовать в CI/CD для проверки инфраструктуры
- Для production окружения рекомендуется использовать `test_qdrant_connection.py` с корректными настройками из `.env`
