# Дизайн: Редизайн подбора апартаментов

> Дата: 2026-03-09
> Статус: Draft
> Scope: telegram_bot/dialogs/funnel.py, telegram_bot/bot.py, telegram_bot/keyboards/

## Проблема

Текущий funnel подбора апартаментов:
- 6 шагов до результатов (город → тип → бюджет → предпочтения → summary → результаты)
- Summary требует выбор "Списком/Карточками" — лишний шаг
- Показывает только top-10 через vector search, а не все подходящие
- Пагинация через inline "Показать ещё" — footer-сообщения плодятся
- Нижнее ReplyKeyboard не меняется при просмотре — бесполезные кнопки на экране
- Нельзя быстро изменить один фильтр без перезапуска из результатов
- Несогласованность page size: funnel = 10, callback = 5
- Счётчик "Найдено X" только после всех шагов

## Решение

### Общая концепция

Упрощённый funnel (3 обязательных + опциональные фильтры) → summary с подтверждением →
карточки пачками по 10 с накоплением в чате → нижнее ReplyKeyboard заменяется на
навигацию каталога → inline фильтр-панель для изменения параметров.

### Flow

```
Кнопка "🏠 Подобрать квартиру"
    │
    ▼
Шаг 1: Город (Select)
    │   Солнечный берег / Свети Влас / Элените / Любой
    ▼
Шаг 2: Комнаты (Select)
    │   Студия / 1-спальня / 2-спальни / 3-спальни / Любой
    ▼
Шаг 3: Бюджет (Select)
    │   До 50к / 50-100к / 100-150к / 150-200к / 200к+ / Любой
    ▼
Шаг 4: "Желаете уточнить?" (Multiselect категорий)
    │   Этаж / Вид / Площадь / Комплекс / Мебель / Акции
    │   или "Нет, перейти к результатам"
    │   → каждая категория открывает под-Select (как сейчас)
    ▼
Шаг 5: Summary + подтверждение
    │   Показывает выбранные фильтры + счётчик "Найдено: X"
    │   [🔍 Найти]  [✏️ Изменить]
    ▼
Шаг 6: Результаты
    │   10 карточек в чат
    │   ReplyKeyboard заменяется на навигацию каталога
    │   Пользователь жмёт "Показать ещё 10" → карточки копятся
    ▼
Выход: "🏠 Главное меню" → возврат обычного ReplyKeyboard
```

### Summary (Шаг 5)

Одно inline-сообщение:

```
🏠 Ваш запрос:
📍 Солнечный берег
🛏 1-спальня
💰 50 000 – 100 000 €
🌅 Вид: Море

Найдено: 23 апартамента
Сортировка: по цене ↑

[🔍 Найти]    [✏️ Изменить]
```

- **Найти** (зелёная, Bot API 9.4) — осознанное подтверждение перед потоком карточек
- **Изменить** — возврат к шагам funnel (дешевле чем менять из результатов)
- Счётчик вычисляется через `Qdrant count(filter)` — быстрый запрос без vector search

### Навигация каталога (ReplyKeyboard)

При входе в результаты нижнее меню заменяется:

```
[📥 Показать ещё 10]  [10 из 47]
[🔍 Фильтры]          [🏠 Главное меню]
```

| Кнопка | Действие |
|--------|----------|
| 📥 Показать ещё 10 | Следующие 10 карточек в чат, счётчик обновляется |
| 10 из 47 | Информационная (кнопка-заглушка или обновляющийся текст) |
| 🔍 Фильтры | Открывает inline фильтр-панель (edit одного сообщения) |
| 🏠 Главное меню | Выход из каталога, возврат обычного ReplyKeyboard |

Когда все показаны: кнопка меняется на `[✅ Все 47 показаны]`.

### Inline фильтр-панель

Одно сообщение, редактируется при каждом изменении:

```
🏠 Поиск апартаментов

Город: Солнечный берег
Комнаты: 1-спальня
Бюджет: 50 000 – 100 000 €

Найдено: 23 апартамента

[🏙 Город ▼]  [🛏 Комнаты ▼]  [💰 Бюджет ▼]
[🌅 Вид ▼]   [📐 Площадь ▼]  [🏢 Этаж ▼]
[🏘 Комплекс ▼] [🛋 Мебель ▼] [🏷 Акции ▼]

[🔍 Применить (23)]                    ← зелёная (Success)
[🗑 Сбросить фильтры]                   ← красная (Danger)
[↩️ Назад к результатам]
```

При нажатии на фильтр — то же сообщение edit:
```
Выберите комнаты:

[  Студия  ] [✅ 1-спальня]
[  2-спальни] [  3-спальни ]
[  Любой   ]

[« Назад]
```

Счётчик "Найдено: X" обновляется при каждом изменении фильтра через `Qdrant count()`.

При нажатии "Применить":
- Если фильтры изменились — очистить предыдущие карточки (новое сообщение-разделитель),
  отправить первые 10 по новым фильтрам
- Если не изменились — вернуть к текущей позиции

### Карточка апартамента

Каждая карточка — отдельное сообщение (фото-альбом + текст + кнопки):

```
📸 [Фото альбом — до 10 фото]

🏠 Premier Fort Beach · Секция A · №42
🛏 1-спальня · 3 этаж · 52 м² · Море
💰 67 000 €

[📌 В избранное]  [💬 Менеджеру]
           [📅 На осмотр]
```

| Кнопка | Стиль | Действие |
|--------|-------|----------|
| 📌 В избранное | Success (зелёная) | Toggle: добавить/убрать из избранного |
| 💬 Менеджеру | Primary (синяя) | Свободный вопрос по этому апарту |
| 📅 На осмотр | Primary (синяя) | Phone collector FSM → менеджеру |

Кнопки работают всегда — карточки остаются в чате, пользователь может вернуться к любой.

### Поиск и сортировка

- **Метод:** чистая payload-фильтрация через Qdrant `scroll()` (не vector search)
- **Сортировка:** по цене ↑ (`order_by: price_eur asc`)
- **Дедупликация:** boundary IDs на границе цен (текущий механизм)
- **Page size:** 10 (единый, без разночтений)
- **Счётчик:** `Qdrant count(filter)` при каждом изменении фильтров

### Новые фичи Telegram (Bot API 9.4-9.5)

| Фича | Применение |
|------|-----------|
| Цветные кнопки (9.4) | Найти/Применить = зелёная, Сбросить = красная, навигация = синяя |
| Кастомные эмодзи на кнопках (9.4) | Иконки 🏙🛏💰🌅📐🏢 на кнопках фильтров |
| CopyTextButton (8.0) | "Скопировать" на карточке (адрес + описание) |

## Что убираем

- ❌ Шаг "Списком/Карточками" — всегда карточки
- ❌ Inline-пагинация "Показать ещё (N осталось)" на footer-сообщениях
- ❌ Кнопка "Запись на осмотр" в footer (она на каждой карточке)
- ❌ Vector search для каталога (только payload filter + scroll)
- ❌ Два page size (10 и 5) — единый = 10

## Что сохраняем

- ✅ 3 обязательных шага funnel (город, комнаты, бюджет)
- ✅ Опциональные предпочтения (вид, этаж, площадь, комплекс, мебель, акции)
- ✅ Summary с обзором выбранного
- ✅ Карточки с фото-альбомом
- ✅ 3 кнопки на карточке (избранное, менеджер, осмотр)
- ✅ Scroll-пагинация по цене через Qdrant (start_from + boundary dedup)
- ✅ Избранное (Postgres CRUD)
- ✅ Phone collector FSM для записи на осмотр
- ✅ Lead scoring (фоновый persist)
- ✅ Langfuse observability

## Что меняем

| Было | Стало |
|------|-------|
| 6 шагов до результатов | 5 шагов (убран выбор режима) |
| Summary: "Списком/Карточками/Изменить/Доп." | Summary: "Найти/Изменить" |
| Inline footer с "Показать ещё" | ReplyKeyboard "Показать ещё 10" |
| Нижнее меню не меняется | Нижнее меню = навигация каталога |
| Page size: 10 (funnel) + 5 (callback) | Единый page size = 10 |
| Vector search (hybrid RRF + ColBERT) | Payload filter + scroll (каталог) |
| Фильтры только через перезапуск funnel | Inline фильтр-панель из результатов |
| Счётчик только после поиска | Счётчик на summary + обновляется при смене фильтров |
