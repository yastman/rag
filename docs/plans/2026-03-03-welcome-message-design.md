# Welcome Message Redesign

**Дата:** 2026-03-03
**Ветка:** `worktree-welcome-message`
**Scope:** Welcome-сообщение + ReplyKeyboard + "Задать вопрос" second-level menu

---

## Проблема

Текущее welcome-сообщение (`/start`) — маркетинговый лендинг (7 буллетов про цены,
рассрочку, паспорт). Нет персонализации, не объясняет что бот умеет, не помогает
начать разговор. По best practices 2026: 68% юзеров завершают разговор если вовлечены
первым сообщением, +40% engagement от персонализации.

## Решение

### 1. Welcome-текст (i18n, 3 локали)

```
Привет, {name}! 👋

Я бот FortNoks Estate — помогу с недвижимостью
в Болгарии. Вот что умею:

🏠 Подобрать апартаменты по вашим параметрам
🔑 Рассказать об услугах агентства
📅 Записать на осмотр или инфотур
👤 Связать с менеджером
💬 Ответить на вопросы о покупке, документах,
   ценах и жизни в Болгарии

Выберите действие внизу или просто напишите вопрос 👇
```

- Персонализация: `{name}` из `message.from_user.first_name`
- Без inline-кнопок в welcome — ReplyKeyboard persistent, не дублируем
- ~45 слов, 5 возможностей

### 2. ReplyKeyboard (обновлённый, 3x2)

```
┌──────────────────────┬──────────────────────┐
│ 🏠 Подобрать квартиру │ 🔑 Услуги            │
├──────────────────────┼──────────────────────┤
│ 📅 Запись на осмотр   │ 👤 Менеджер          │
├──────────────────────┼──────────────────────┤
│ 💬 Задать вопрос      │ 📌 Мои закладки      │
└──────────────────────┴──────────────────────┘
```

**Diff vs текущий:**

| Было | Стало |
|------|-------|
| 🏠 Подбор апартаментов | 🏠 Подобрать квартиру |
| 🔑 Услуги | 🔑 Услуги (без изменений) |
| 📅 Запись на осмотр | 📅 Запись на осмотр (без изменений) |
| 📌 Мои закладки | 📌 Мои закладки (переместили в row 3) |
| 🎁 Акции | **УБРАНА** (не core, доступна контекстно) |
| 👤 Связь с менеджером | 👤 Менеджер (короче) |
| — | 💬 Задать вопрос (**НОВАЯ**) |

### 3. "💬 Задать вопрос" — second-level inline menu

При нажатии кнопки "💬 Задать вопрос" бот отправляет сообщение с InlineKeyboard:

```
💬 Напишите вопрос — мы с радостью ответим!

Или выберите популярную тему:

[📋 Какие документы нужны для покупки?]
[💰 Сколько стоит оформление сделки?]
[📋 Как получить ВНЖ в Болгарии?]
[💳 Какие условия рассрочки?]
```

**Callback маппинг:**

| Кнопка | Callback data | Действие |
|--------|---------------|----------|
| Документы для покупки | `ask:docs` | `handle_query("Какие документы нужны для покупки?")` |
| Стоимость оформления | `ask:costs` | `handle_query("Сколько стоит оформление сделки?")` |
| ВНЖ в Болгарии | `ask:vnzh` | `handle_query("Как получить ВНЖ в Болгарии?")` |
| Условия рассрочки | `ask:installment` | `handle_query("Какие условия рассрочки?")` |

Вопросы подобраны из FAQ_PATTERNS в `classify.py` — самые частые темы покупателей.
Каждая кнопка прокидывает текст в RAG pipeline через `handle_query`. Без нового кода
логики — только маппинг callback → текст.

## Не затрагивается

- RAG pipeline, classify_node, graph — без изменений
- Существующие handlers (`_handle_search`, `_handle_services`, `_handle_viewing`,
  `_handle_manager`, `_handle_bookmarks`) — переиспользуются as-is
- aiogram-dialog dialogs (FunnelSG, FaqSG, etc.) — без изменений

## Scope 2 (отдельный PR, не в этом)

- Post-click UX: промежуточные объяснения после нажатия "Менеджер" и "Осмотр"
- Акции: контекстное отображение вместо кнопки в меню

## Принципы (best practices 2026)

- 20-50 слов в welcome
- Max 3 inline-кнопки в одном сообщении (ask menu = 4, допустимо для second-level)
- Персонализация по имени (+40% engagement)
- Кнопки с quick actions (+35-50% engagement vs open-ended)
- Один набор навигации (ReplyKeyboard), inline только для контекстных действий
