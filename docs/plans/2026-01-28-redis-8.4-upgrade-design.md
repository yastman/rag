# Redis 8.4 Upgrade Design

**Date:** 2026-01-28
**Status:** Draft
**Author:** Claude + User

## Summary

Upgrade Redis from `redis/redis-stack:latest` to `redis:8.4.0` for improved performance, stability, and native Query Engine support.

## Current State

```yaml
# docker-compose.dev.yml, docker-compose.local.yml
image: redis/redis-stack:latest
```

**Problems:**
- Unpinned version — can break on auto-update
- Redis Stack 7.4 is in maintenance mode
- Missing Redis 8.x performance improvements (+87% faster commands)

## Target State

```yaml
image: redis:8.4.0
```

**Benefits:**
- Redis Query Engine built-in (FT.* commands)
- FT.HYBRID — native RRF fusion
- SIMD acceleration for vector operations (AVX512)
- MSETEX — atomic multi-key TTL
- BM25STD as default scorer
- +87% faster commands vs Redis 7

## Implementation Plan

### Step 1: Update Docker Compose Files

**Files to modify:**
- `docker-compose.dev.yml`
- `docker-compose.local.yml`

**Changes:**
```yaml
# Before
redis:
  image: redis/redis-stack:latest
  ports:
    - "6379:6379"
    - "8001:8001"  # RedisInsight (Stack only)

# After
redis:
  image: redis:8.4.0
  ports:
    - "6379:6379"
  command: >
    redis-server
    --loadmodule /opt/redis-stack/lib/redisearch.so
    --loadmodule /opt/redis-stack/lib/rejson.so
```

**Note:** Redis 8.4 OSS includes Query Engine natively — no module loading required for FT.* commands. JSON module may need separate consideration if used.

**Simplified version (if only FT.* needed):**
```yaml
redis:
  image: redis:8.4.0
  ports:
    - "6379:6379"
```

### Step 2: Add Smoke Test to Makefile

**File:** `Makefile`

```makefile
.PHONY: test-redis
test-redis: ## Verify Redis Query Engine is available
	@echo "Testing Redis Query Engine..."
	@docker exec $$(docker ps -qf "name=redis") redis-cli FT._LIST || (echo "FAIL: FT._LIST not available" && exit 1)
	@docker exec $$(docker ps -qf "name=redis") redis-cli FT.CREATE test_idx ON HASH PREFIX 1 test: SCHEMA name TEXT || (echo "FAIL: Cannot create index" && exit 1)
	@docker exec $$(docker ps -qf "name=redis") redis-cli FT.DROPINDEX test_idx || true
	@echo "Redis Query Engine OK"
```

### Step 3: Add CI Verification

**File:** `.github/workflows/ci.yml` (or equivalent)

```yaml
- name: Verify Redis Query Engine
  run: |
    docker exec redis redis-cli FT._LIST
    docker exec redis redis-cli FT.CREATE test_idx ON HASH PREFIX 1 test: SCHEMA name TEXT
    docker exec redis redis-cli FT.DROPINDEX test_idx
```

## Rollback Plan

If issues occur:

```yaml
# Revert to previous image
image: redis/redis-stack:7.4.0-v0
```

## Testing Checklist

- [ ] `docker-compose up -d redis` starts successfully
- [ ] `redis-cli FT._LIST` returns empty list (no error)
- [ ] `redis-cli FT.CREATE` creates index
- [ ] `redis-cli FT.SEARCH` works
- [ ] RedisVL SemanticCache initializes
- [ ] Semantic cache hit/miss works
- [ ] Embeddings cache works
- [ ] All unit tests pass
- [ ] All integration tests pass

## Risk Assessment

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| FT.* commands not available | Low | High | Smoke test in CI |
| RedisVL incompatibility | Low | Medium | Test before deploy |
| Data loss on upgrade | None | N/A | Cache is ephemeral |
| Performance regression | Very Low | Low | Redis 8.4 is faster |
