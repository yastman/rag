# Documentation System Design

**Date:** 2026-01-26
**Status:** Approved
**Author:** Claude Code + Human

---

## Problem Statement

1. **Session amnesia** - Each Claude Code session starts fresh, requiring "exploring codebase" every time
2. **Bloated TODO.md** - 169 lines of completed tasks mixed with active ones
3. **Manual CHANGELOG** - 416 lines maintained by hand, error-prone
4. **No persistent context** - No way to track "what we're working on now"

---

## Solution Overview

```
Система "Обнуление + История" (Best Practice 2026)

TODO.md (живой)     →    Git commits (история)    →    CHANGELOG.md (авто)
    ↓                          ↓                              ↓
 Что делать              Conventional Commits           Что сделали
 сейчас (15 строк)       feat:, fix:, docs:              по версиям
```

## Component Details

### 1. TODO.md (New Format)

```markdown
# TODO

## In Progress
- [ ] feat(search): add binary quantization A/B testing

## Next Up
- [ ] fix(cache): Redis connection pooling
- [ ] feat(bot): voice message support
- [ ] docs: update API documentation

## Ideas (Backlog)
- Admin panel
- Prometheus metrics
- Multi-language support

---
*Tasks completed? Delete them. History lives in git commits.*
```

**Rules:**
- Max 20 lines
- When task is done → delete it (not mark as done)
- History is in git log: `git log --oneline --grep="feat\|fix"`

### 3. Release Please Configuration

**release-please-config.json:**
```json
{
  "$schema": "https://raw.githubusercontent.com/googleapis/release-please/main/schemas/config.json",
  "release-type": "python",
  "packages": {
    ".": {
      "changelog-path": "CHANGELOG.md",
      "release-type": "python",
      "bump-minor-pre-major": true,
      "bump-patch-for-minor-pre-major": true,
      "changelog-sections": [
        {"type": "feat", "section": "Features", "hidden": false},
        {"type": "fix", "section": "Bug Fixes", "hidden": false},
        {"type": "perf", "section": "Performance", "hidden": false},
        {"type": "docs", "section": "Documentation", "hidden": true},
        {"type": "chore", "section": "Miscellaneous", "hidden": true},
        {"type": "refactor", "section": "Code Refactoring", "hidden": true},
        {"type": "test", "section": "Tests", "hidden": true}
      ]
    }
  }
}
```

**.release-please-manifest.json:**
```json
{
  ".": "2.12.0"
}
```

**.github/workflows/release-please.yml:**
```yaml
name: Release Please

on:
  push:
    branches:
      - main

permissions:
  contents: write
  pull-requests: write

jobs:
  release-please:
    runs-on: ubuntu-latest
    steps:
      - uses: googleapis/release-please-action@v4
        id: release
        with:
          config-file: release-please-config.json
          manifest-file: .release-please-manifest.json

      # Outputs for downstream jobs
      - name: Output release info
        if: ${{ steps.release.outputs.release_created }}
        run: |
          echo "Release created: ${{ steps.release.outputs.tag_name }}"
          echo "Version: ${{ steps.release.outputs.version }}"
```

**Workflow:**
1. Push to `main` with conventional commits
2. Release Please creates/updates PR with:
   - Updated CHANGELOG.md
   - Bumped version in manifest
3. Merge PR → Release Please creates git tag
4. Tag triggers existing `release.yml` → Deploy

### 4. Commit Message Convention (Already in Use)

```
<type>(<scope>): <subject>

Types:
- feat:     New feature (→ CHANGELOG)
- fix:      Bug fix (→ CHANGELOG)
- perf:     Performance improvement (→ CHANGELOG)
- docs:     Documentation only
- refactor: Code refactoring
- test:     Adding tests
- chore:    Maintenance (deps, configs)

Examples:
feat(search): add binary quantization support
fix(cache): resolve Redis connection timeout
perf(embedding): implement batch processing
docs: update CLAUDE.md with current sprint
```
