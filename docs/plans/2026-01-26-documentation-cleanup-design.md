# Documentation Cleanup Design

**Date:** 2026-01-26
**Status:** Implemented

## Problem

Documentation sprawl:
- 20 markdown files in root directory
- Outdated versions everywhere (2.0.1, 2.9.0, 2.12.0 vs actual 2.13.0)
- Duplicate content (README.md vs docs/README.md)
- Broken links in docs/index.md
- No single source of truth

## Decision

Minimize to essentials. CLAUDE.md is the primary context for Claude Code CLI workflow.

## Target Structure

### Root (4 files)

```
README.md        # 35 lines, badges + links to CLAUDE.md
CLAUDE.md        # Full technical context (already maintained)
CHANGELOG.md     # Auto-generated by Release Please
TODO.md          # Current sprint tasks
```

### docs/

```
docs/
├── PIPELINE_OVERVIEW.md    # Architecture (keep)
├── QDRANT_STACK.md         # Qdrant config (keep)
├── LOCAL-DEVELOPMENT.md    # Dev setup (keep)
├── plans/                  # Design documents
├── archive/                # Historical docs
│   └── 2026-01-26-cleanup/ # This cleanup
├── documents/              # Source DOCX files
├── processed/              # Processed data
└── test_data/              # Test fixtures
```

## Rationale

1. **CLAUDE.md is king** — Claude Code reads it automatically, always up-to-date
2. **README.md is a landing page** — badges, quick start, links to CLAUDE.md
3. **CHANGELOG.md is automated** — Release Please handles versioning
4. **Archive, don't delete** — historical context preserved but out of the way

## Maintenance

- Update CLAUDE.md when architecture changes
- CHANGELOG.md auto-updates on release
- Archive old plans after completion
